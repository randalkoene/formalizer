Tips

  1. Download the presentation as PDF.
  2. Move the PDF to a new folder to keep related things in one place.
  3. In that folder, use `thumbpdf` to create the thumbnail images. See example below and notes.

Example:

		thumbpdf --nomakepdf onboarding-slides.pdf

Notes:

  • Without `--nomakepdf` the thumbnails are placed into a new PDF and the image files are deleted.
  • You may need to rename the PDF, because `thumbpdf` is pretty finicky about file names.
  • The thumbnails may be too small at the default resolution (9 == 90 pixels wide). For example, try `--resolution 32` (320px wide) or `--resolution 64` (640px wide).

The easiest solution is (on a full Ubuntu system, like alcyone) to press: CTRL+ALT+SHIFT+R

That is the Gnome Recorder. You can pick a region on the screen, or a window, or a screen. The biggest issue with this one is that it does not record sound. Recordings are of webm type, which isn't really an issue most of the time.

For video+audio, you need to use the EasyScreenCast Gnome extension. For this, you need to install the Gnome Extension Manager (available through the App Center). Restart the Gnome shell (ALT+F2 then type r in X, in Wayland you have to log out and back in). Once installed, the built-in recording capability can also handle audio sources. You can click on the camera icon in the control bar and select and audio source there.

Note: I found that invoking Gnome Screen Recorder with the key combination still invokes recording without sound. Use the camera icon at top to set up what to record and start recording instead. The result by default ends up in ~/Videos. This was tested to work on alcyone. I found that selecting stereo analog headphone audio as the source worked well, while selecting default audio did not. Test before the session to be recorded, and make sure you don't get duplicate sound when attending from another laptop, i.e. disconnect the target end of the audio cable so that audio still goes there but doesn't arrive anywhere physical.
