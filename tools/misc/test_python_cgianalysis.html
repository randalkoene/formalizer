CGI Call:

Note that the call entered is performed from within the context of Python subprocess calls on the OS. The path must therefore be one of these:

  • Absolute, e.g: /usr/lib/cgi-bin/test_python_cgisubprocess.py
  • Relative, e.g: ./test_python_cgisubprocess.py
  • Discoverable on a minimal PATH: date
(The PATH is typically something like: /usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/snap/bin)

Important: The cgi-www user cannot write to the /var/www/html/formalizer directory tree. This means that any claims that error output is being written to a file there are false. Instead, try redirecting error output to STDOUT, e.g.: ./fzloghtml -E STDOUT -d formalizer -s randalk
