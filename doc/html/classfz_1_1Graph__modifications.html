Detailed Description

This data structure is the most efficient method to request a stack of Graph modifications.

It is constructed in shared memory using the allocate_Graph_modifications_in_shared_memory() below and given to fzserverpq. See fzaddnote for an example of its use.

All requested Graph modifications are in relation to an existing Graph, which is referenced in shared memory at the address in the graph_ptr cached pointer. Note that this variable is not a offset_ptr, because it is not pointing to an object in the same segment. The fzserverpq does not rely on this value to apply the modification requests.

Creating this object will throw a Shared_Memory_Exception if no shared memory containing a Graph is active.
