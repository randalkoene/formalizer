Detailed Description

Specifies the start time, Node, close time, and set of entries of a Log chunk.

Note A: The Log_chunk_ID differs from Log_entry_ID in that there is no minor_id.

Note B: It is possible to get the sorted list of Log chunks that belong to a specific Node by parsing the list log Log chunks and identifying all of the ones that belong to the Node. Not only would that be slow (which would at least sugegst maintaining a rapid-access list), but it is also counter-productive when you expect that you will rarely load the complete Log into memory at once. For this reason, Log_chunk also explicitly stores node_prevchunk_id and node_nextchunk_id.
