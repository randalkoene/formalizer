template<class Log_ID_key, class Log_component>
struct fz::Log_target< Log_ID_key, Log_component >

Log objects are principally identified by their ID key, but when used as a linked target, e.g.

by-Node chaining, rapid-access pointers are maintained when possible for faster traveral. This Log_target template formalizes that concept.

*** Note: In a future update, this template can be the basis for added pointer safety with regards to rapid-access pointers. See the description on the Trello card at https://trello.com/c/CZ8XIw4j.
